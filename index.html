Status — v10 Is Effectively a v9 Re-Run. The Corpus Expansion Hypothesis Was Not Tested.

Update 2026-04-30: v10_dialog_corpus deployed live — 384D / 12L / 12H / RoPE / Pre-LN / AdamW, vocab=9,509, brain_params=21,509,761. Trained 2026-04-29 08:52 UTC on Modal L4, ~$4.

The bug that invalidated v10 as an experiment: v10 was launched on Modal with BOOKS=["corpus/dailydialog_clean.txt", "corpus/everyday_conversations.txt"], but corpus/everyday_conversations.txt was not committed to GitHub at the time of launch. Modal training clones the repo from GitHub — the file was not in the container. ingest.py line 134 silently filters missing paths: corpus_files = [b for b in args.books if os.path.exists(b)]. Result: v10 trained on dailydialog_clean.txt only — the exact same corpus as v9. Confirmed by vocab_size=9,509 — bit-identical to v9. A real corpus expansion would have changed it. v10 is effectively a v9 re-run with a different random seed. $4 of Modal time was wasted.

Eval results (eval_chat.py, 16 prompts — eval_runs/chat_20260429_200824.json): Criterion 1 (distinct outputs) — PASS, 16/16 distinct. Criterion 2 (≥80% real-English tokens) — mostly pass, real English words but with odd tokens like kramer, wisconsin, interpersonal. Criterion 3 (≥3/5 chitchat replies coherent) — FAIL, still word salad. Fragments are marginally more grammatical than v9 ("you re a", "i don t", "i ve have", "i m a") but no coherent sentences.

Sample replies from eval transcript (eval_runs/chat_20260429_200824.json):

Lesson logged in plans/V11_PLAN.md pre-flight checklist: "BOOKS files must be in HEAD before launch; verify with git ls-tree HEAD corpus/." Next concrete step is a v10-redo — commit everyday_conversations.txt first, then re-run with both corpora actually present. That will finally test the original corpus-expansion hypothesis. v11 corpus expansion (smol-smoltalk, MultivexAI) is on hold — smol-smoltalk proved to be instruction-tuning data (system prompts, code generation) rather than conversational; MultivexAI was only 30 KB of single-sentence utterances. Neither is suitable. v11 corpus selection is TBD pending the v10-redo result.

Evaluation Report — Human Judgement

A 16-prompt evaluation across three difficulty buckets (greetings · chitchat · harder Q&A) was run against v4_rope. Result: all 32 generated replies were near-identical sequences of rare/odd words regardless of input prompt. Example prompt/reply pairs:

[USER] hello
[BOT] you for suavely user resigning to the for impressiveness resigning i you freshener that the

[USER] what is the capital of france
[BOT] that slighted resigning to impressiveness you the i and to resigning the defilements for you

[USER] tell me a story
[BOT] you for suavely user resigning to the for impressiveness resigning i you freshener that the

Diagnosis: the model isn't conditioning on input prompts. The output collapses to the same vocab cluster regardless of context. Two plausible causes:

  • Undertraining — 1 epoch on a corpus where ~50% is 19th-century novels means rare archaic vocab dominates the output distribution. Classic sign of insufficient gradient steps.
  • Eval bug — the bench harness used torch.randn() embeddings in its eval_cosine_top1, which never moved off 5.0% across any architectural change. The metric may be saturated / noise-dominated, but live eval on real ChromaDB embeddings still produces nonsense — so undertraining is the dominant explanation.
