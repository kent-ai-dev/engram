Status — v13_xent_temp30: Sharper Temperature Dropped Loss 2.34 Nats. Output Is Still Token-Soup. Frozen Vocab Is Now the Suspect.

Update 2026-05-04: v13_xent_temp30 deployed live — single-variable swap from v12: INV_TEMPERATURE raised from 10 to 30 (predicted_norm @ vocab_matrix_normed.T * INV_TEMP=30). Architecture and corpus are identical to v12 (12 layers, 384D, 12H, RoPE, Pre-LN, AdamW, ~21.5M brain / ~38.7M engram params, vocab 14,704, context 32, 5.3 MB corpus). Trained 2026-05-04 on Modal L4, $6.00.

Loss result: final loss 5.04 nats (vs v12's 7.38) — a 2.34 nat absolute reduction. Theoretical floor at INV_TEMP=30: ≈0.59 nats (dropped 1.18 nats from v12's ≈1.77). Net headroom-above-floor narrowed only ~1.16 nats (5.6 → 4.45). Convergence remained healthy — the sharper temperature did exactly what it was supposed to do for the loss signal.

Eval verdict — PARTIAL/FAIL. Output is incoherent. Replies differ per prompt (distinct criterion: PASS). avg_ponder consistently 2.7–2.9 (halt-gate cap=3), indicating the halt gate is near saturation — the model is using almost all available pondering steps before halting. But output is token-soup — not English sentences. Coherence did not change versus v12.

Diagnosis: the brain can reduce loss via sharper temperature — it is genuinely getting better gradient signal. But the predicted concept vectors are not landing on coherent ChromaDB tokens. The vocab embeddings are frozen: initialized once from the teacher model and never updated during training. A more confident brain pointing in random directions of a frozen vocab space produces no better text. The sharpened softmax is pushing harder on a target that was never trained to align with it.

What this rules out: temperature calibration was a real bottleneck for the loss metric, but it is not the bottleneck for coherence. The fundamental issue is that the vocab embedding matrix is static. The model brain learns to predict concept vectors; those vectors are decoded via nearest-neighbor lookup against frozen embeddings. If the frozen embeddings do not form a geometry that the brain's predictions can converge on, no amount of temperature adjustment will produce coherent tokens.

Next — v14 Branch B: make the ChromaDB vocab a learnable nn.Parameter. This is the architectural change implied by the v13 result. Branch A (lift halt-gate cap from 3 to a higher value, given saturated avg_ponder) is queued as a parallel signal. See plans/V14_CANDIDATES.md for the full decision tree (branches A/B/C/D) and plans/FUTURE_RESEARCH.md for the v15+ backlog. Cumulative spend: ~$57 of $150 budget ceiling.

Evaluation Report — Human Judgement

A 16-prompt evaluation across three difficulty buckets (greetings · chitchat · harder Q&A) was run against v4_rope. Result: all 32 generated replies were near-identical sequences of rare/odd words regardless of input prompt. Example prompt/reply pairs:

[USER] hello
[BOT] you for suavely user resigning to the for impressiveness resigning i you freshener that the

[USER] what is the capital of france
[BOT] that slighted resigning to impressiveness you the i and to resigning the defilements for you

[USER] tell me a story
[BOT] you for suavely user resigning to the for impressiveness resigning i you freshener that the

Diagnosis: the model isn't conditioning on input prompts. The output collapses to the same vocab cluster regardless of context. Two plausible causes:

  • Undertraining — 1 epoch on a corpus where ~50% is 19th-century novels means rare archaic vocab dominates the output distribution. Classic sign of insufficient gradient steps.
  • Eval bug — the bench harness used torch.randn() embeddings in its eval_cosine_top1, which never moved off 5.0% across any architectural change. The metric may be saturated / noise-dominated, but live eval on real ChromaDB embeddings still produces nonsense — so undertraining is the dominant explanation.

Logical Next Steps (updated 2026-05-04)
  1. v14 Branch B — make ChromaDB vocab a learnable nn.Parameter. v13 demonstrated that a more confident brain pointing into a frozen vocab space produces no improvement in coherence. The diagnosis is clear: the vocab embedding matrix is initialized once from the teacher and never updated. Making it an nn.Parameter allows the token space to co-adapt with the brain's predictions during training. This is the architectural break the v13 result points to.
  2. v14 Branch A — lift the halt-gate cap above 3. avg_ponder was consistently 2.7–2.9 across v13 evals, indicating the gate is nearly saturated at the cap=3 ceiling. This is an independent signal worth testing in parallel: if the model wants to ponder more steps but is being cut off, we may be artificially limiting its reasoning depth. This is a cheaper experiment than Branch B and can run concurrently. See plans/V14_CANDIDATES.md for branches C and D as well.
  3. Remaining variables to defer. Four levers have been tested and refuted (capacity, corpus volume, loss function class, temperature calibration). Context length and architecture depth are in the research backlog (plans/FUTURE_RESEARCH.md) but should not be touched until the vocab-learnability and halt-gate hypotheses are resolved.
